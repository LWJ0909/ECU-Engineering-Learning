# ECU 개발용 RC카 제작: 하드웨어 선정 과정의 어려움

## 📅 날짜
2025년 1월 19일

## 🎯 프로젝트 목표
ECU(Electronic Control Unit) 개발 학습을 위한 라즈베리파이 5 + STM32 기반 RC카 제작

> **v1.0 단계**: 하드웨어는 라즈베리파이 5 기준으로 먼저 구성하고,  
> 이후 STM32를 추가해서 듀얼 ECU 구조로 확장할 계획이다.

---

## 😰 실제로 겪었던 어려움들 (시간 순서대로)

### 1. 하드웨어 사는 것부터가 어려웠다

**가장 근본적인 문제:**
- 뭘 사야 하는지부터 막막함
- 검색해도 제품이 너무 많고 용어가 낯섦
- "이걸 사면 되는 건가?" 확신이 안 섬

---

### 2. 원하는 키트가 존재하지 않았다

**초기 계획:**
```
"라즈베리파이5와 STM32를 모두 장착할 수 있는
완성된 키트를 사면 편하지 않을까?"
```

**현실:**
- 시중에 있는 건 대부분 **아두이노 전용** 또는 **라즈베리파이 온리** 교육용 키트
- 라파+STM32 듀얼 구성 키트는 거의 없음
- 교육용 키트들은:
  - ❌ 내게 필요 없는 기능이 너무 많음 (예: 라인트레이서 센서, LED 등)
  - ❌ 확장이 어려움 (고정된 구성)
  - ❌ ECU 개발 학습용으로는 부적합

**결론:**
```
→ 차라리 "프레임(섀시)"만 사서
→ 내가 원하는 부품을 직접 조립하는 게 낫겠다!
```

**깨달은 점:**
- 완성 키트 ≠ 유연성
- ECU 개발 학습에는 커스텀 구성이 필수
- 프레임+모터만 있는 베이스 섀시가 오히려 좋음

---

### 3. 부품 간 호환성 - 가장 큰 무지

**핵심 문제:**
> "배선부터 하는 하드웨어 설계는 처음이다 보니  
> 어떤 부품이 서로 호환이 되는지에 대한 무지가 컸다"

**구체적 혼란:**

#### 전압 호환성
```
섀시 모터: 12V
배터리: 3S = 11.1V
→ 이게 호환돼?
→ "범위"가 있다는 걸 몰랐음
```

#### 전류 용량
```
모터 드라이버: 2A
모터: 실제 몇 A 먹는지 모름
→ 부족하면? 타버리나?
```

#### 인터페이스 호환성
```
라즈베리파이 GPIO ↔ L298N
→ 어떻게 연결하지?
→ 전압 레벨이 다르면?
```

**해결 과정:**
- AI한테 계속 물어봄
- "이거랑 이거 호환돼?" 식으로 확인
- 시스템 전체 구성도 그려서 이해

**배운 것:**
- 전압 "범위" 확인 (정확히 같을 필요 없음)
- 전류는 여유있게 (모터 드라이버가 모터보다 용량 커야)
- 제어 신호는 3.3V/5V 로직 레벨 확인
- 공통 그라운드(GND) 연결 필수!

---

### 4. 납땜을 피하고 싶었는데, 어디서 피할 수 있는지 불명확

**목표:**
- 가능한 한 납땜 없이 조립하고 싶음
- 납땜 경험 없음

**헷갈린 부분:**
- 어떤 부품은 납땜 필수인가?
- 어떤 부품은 안 해도 되나?
- 터미널 블록, 듀폰 케이블 같은 용어가 낯섦

**AI 설명 듣고 정리:**

#### 납땜 필요 없는 방식:
- **터미널 블록** (나사로 조임) ← L298N 모터 연결
- **듀폰 케이블** (핀에 꽂기) ← GPIO 연결
- **XT60 커넥터** (플러그 방식) ← 배터리 연결
- **USB 케이블** ← 전원 공급

#### 납땜 필요한 경우:
- XT60 커넥터를 케이블에 직접 연결할 때
- 커스텀 배선할 때

**결론:**
- 모듈 형태 부품 선택하면 대부분 납땜 불필요!
- L298N, DC-DC 컨버터 모두 터미널 블록 있는 제품 선택
- 듀폰 케이블 세트 구매하면 GPIO 연결 쉬움

---

### 5. 섀시에 모터가 포함인지 아닌지 모름

**문제:**
- 섀시 제품 보면서 계속 헷갈림
- "이거 모터 같이 오는 거야? 따로 사야 해?"
- 제품 설명만 봐서는 명확하지 않음

**확인 방법:**
- 제품 사진 올려서 AI한테 물어봄
- 포장 목록(Packing List) 확인
- "인코더 기어 모터 ×2" ← 이런 식으로 명시돼 있어야 함

**배운 것:**
- "Full Kit", "Complete Set" 같은 단어 찾기
- "섀시 온리", "프레임만" 같은 표현 주의
- 리뷰에서 "모터 같이 왔어요" 확인

---

### 6. 모터에 맞는 전원과 충전기 찾기가 어려웠다

**가장 헷갈렸던 부분:**

#### 6-1. 모터 전압 확인
```
섀시: "12V Motor"
→ 배터리는 몇 볼트를 사야 하지?
→ 정확히 12V? 아니면 범위가 있나?
```

**알게 된 것:**
- 모터는 전압 "범위"가 있음 (11V~16V 같은)
- 18650 3S = 11.1V (만충 12.6V) → 12V 모터에 호환!

#### 6-2. 충전기 찾기
```
"18650 3S 배터리" 검색
→ 충전기는?
→ "DC 12.6V 어댑터" 발견
→ 이게 충전기야? 전원 공급기야?
```

**혼란:**
- 처음 본 제품은 전원 어댑터였음 (충전 기능 없음)
- Y자형 배터리팩 + 어댑터 세트를 찾아야 충전 가능
- "BMS 포함", "과충전 보호" 같은 문구 확인 필요

**결론:**
- Y자형 배터리팩 + 전용 충전 어댑터 세트가 답
- 배터리 전압과 모터 전압 범위 확인 필수

---

### 7. L298N 모터 드라이버 - 어떤 게 맞는 건지 헷갈림

**문제:**
- 검색하면 L298N이 여러 종류로 나옴
- "2A", "5A", "SMG", "NEROMART" 등 브랜드도 다양
- 칩 자체도 있고, 모듈도 있고
- **어떤 게 내 프로젝트에 맞는지 모르겠음**

**혼란스러웠던 부분:**
```
1. SMG 2A L298 모터드라이버 모듈
2. STMicroelectronics L298N (칩 자체)
3. SMG L298N 아두이노 Dual H-Bridge 모듈
4. OEM 70W+70W 5A L298N
5. NEROMART Dual DC Motor Module

→ 이 중에 뭘 사야 하지?
```

**AI와 대화로 정리:**
- 칩 자체(2번)는 초보자한테 안 맞음 (회로 설계 필요)
- 모듈 형태(1,3,5번)가 초보자용
- 기본 기능은 다 비슷함
- 1번이나 3번이 표준적이고 저렴

---

### 8. Y자형 vs I자형 배터리 - 처음 들어봄

**문제:**
- 배터리팩 찾다가 "Y자형"이라는 용어 발견
- 이게 뭔지, I자형이랑 뭐가 다른지 전혀 모름

**AI 설명 듣고 이해:**
```
Y자형:
  배터리 ─┬─ 사용 케이블
         └─ 충전 케이블
         
I자형:
  배터리 ── 단일 케이블 (사용/충전 공용)
```

**왜 Y자형 선택:**
- 충전 쉬움 (케이블 분리 안 해도 됨)
- 초보자 친화적
- 충전기 포함된 제품 있음

---

### 9. 전압강하 모듈(DC-DC 컨버터)의 필요성조차 몰랐다

**초기 생각:**
```
"배터리 → 라즈베리파이 연결하면 되는 거 아냐?"
```

**문제:**
- 배터리: 11.1V~12.6V
- 라즈베리파이: 5V 필요
- 직접 연결하면 라즈베리파이 고장!

**AI한테 듣고 알게 됨:**
```
전압강하 모듈(DC-DC 컨버터) 필요!
12V → 5V로 변환해주는 부품

SMG 5V 5A 컨버터 같은 제품
```

**전체 전원 구조 이해:**
```
배터리 12V
  ├─→ 모터 드라이버 (12V 그대로)
  └─→ DC-DC 컨버터 → 5V → 라즈베리파이
```

**배운 점:**
- 전압이 다른 부품들은 각각 맞는 전압으로 변환 필요
- Y케이블로 전원 분기
- 공통 그라운드(GND) 연결 필수

4. **제품 상세 페이지 꼼꼼히 보기**

## 🎓 배운 것들

### 1. 하드웨어 설계의 기본

#### 전압 체크가 모든 것의 시작
- 각 부품의 동작 전압 확인
- 전압 "범위" 이해 (정확히 같을 필요 없음)
- 변환 필요한 곳 파악 (DC-DC 컨버터)

#### 전류 용량
- 모터 드라이버는 모터보다 용량 여유있게
- 배터리 용량(mAh)은 주행 시간 결정

#### 인터페이스
- GPIO: 3.3V/5V 로직 레벨
- 터미널 블록: 나사 조임
- 듀폰 케이블: 핀 꽂기
- XT60: 고전류 커넥터

### 2. 납땜 없이도 가능하다!
- 모듈 형태 부품 선택
- 터미널 블록, 듀폰 케이블, XT60
- 초보자도 조립 가능

### 3. 나만의 구성이 답이다
- 완성 키트: 빠르지만 확장 제한적
- 베이스 섀시 + 직접 구성: 자유롭고 학습에 최적
- ECU 개발에는 커스텀 구성이 필수

### 4. AI 활용이 큰 도움
- 모르는 건 바로 질문
- 제품 사진 올려서 확인
- 단계별로 이해
